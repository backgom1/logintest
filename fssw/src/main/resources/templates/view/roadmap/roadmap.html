Database !
데이터베이스를 한 마디로 정의하면 ‘데이터의 집합’이라고 할 수 있습니다.

데이터베이스에는 일상생활 대부분의 정보가 저장되고 관리됩니다. 오늘 보내거나 받은 카카오톡 메시지, 인스타그램에 등록한 사진, 
버스/지하철에서 찍은 교통카드, 카페에서 구매한 아이스 아메리카노 등의 정보가 모두 데이터베이스에 기록됩니다.

DBMS를 통해 관리하며 학교에서는 관계형 데이터베이스를 위주로 배우며 
SQL문을 통해 데이터를 관리하는 방법을 배웁니다. 프로젝트 진행시 절때 빠질 수 없는 부분이며 데이터들은 ERD를 통해 데이터 모델링을 합니다.



<알면 좋은개념>
DBMS,ERD,SQL(관계형DB), NoSQL(비관계형DB)

팀원들과 협업을 하거나 혼자 프로젝트를 진행할 때 소스 코드를 수정한 내역들을 저장하고 언제든 찾아볼 수 있습니다

Github 라는 사이트를 통해 GUI 환경에서 손쉽게 다룰 수도 있습니다.

이미 여러 회사에서 git을 통해 버전 관리를 하고 있어서 필수적으로 공부해야하는 친구입니다

Java 프로젝트를 진행하면서 Git으로 버전 관리를 하면 1석 2조의 효과를 볼 수 있을거예요^^

Spring !
자바기반으로 만들어진 프레임워크(프로그램의 기초를 구축할 수 있는 코드의 집합)를 통해 제공되는 여러 기능들을 사용해서

빠르고 효율적으로 프로그램 구축 가능. 요즘 웹백엔드계의 취업의 필수 사항으로 학교에서 다루지는 않지만 프로젝트를 진행할때

spring framework를 사용하여 익숙해진다면 좋다. 
여러가지 디자인패턴(설계자들이 올바른 설계를 빠르게 만들 수 있도록 도와줌)을 사용함.

<알면 좋은개념>
디자인 패턴, 프레임워크
JPA !
JPA 는 객체지향 개념을 허용하지 않는 데이터베이스를 객체 지향으로 사용할 수 있게끔 도와주는 기술입니다

데이터베이스를 사용할 때 SQL문이 아닌 METHOD를 이용하여 DB를 조작할 수 있습니다

고로 SQL문을 깊게 고민할 필요 없이 쉽게 예측하여 조작할 수 있습니다

<알면 좋은 개념>
기존의 데이터베이스를 처리하는 방법(JDBC)과 JPA의 차이점, JPA로 얻을 수 있는 장점들
Docker !
Docker란 기존의 운영체제에서 소프트웨어를 실행하며 다루던 방식이 아닌

작은 컨테이너를 이용하여 빠르고 최적화하여 어플리케이션을 테스트, 배포하는 환경을 구축하는 플랫폼입니다

프로그램을 사용하는데 필요한 최소한의 자원들만 가지고 운영하기 때문에 자원을 훨씬 아끼며 환경을 구축할 수 있습니다

사용자들의 요구가 많아지는 세상에서 빠른 속도와 작은 용량을 자연스럽게 제공해주는 점이 아주 매력적입니다

컨테이너 방식은 앞으로 더 많은 곳에서 사용될 수 있는 기술로 보아 꼭 다룰 줄 알아야 하는 기술입니다

Java 프로젝트를 진행하며 Git을 통해 버전 관리를 하고 JPA를 통해 데이터베이스와 연결하고 Docker를 통해 웹 서버를 배포하면 아주 좋은 경험이 완성될 겁니다

<알면 좋은 개념>
Docker의 개념, 컨테이너, 이미지 등과 같은 개념, 컨테이너 방식으로 웹 서버 배포 방법
웹 서버 !
웹 서버란 웹 사이트에 사용자가 접근하려고 할때 요청한 사이트의 페이지, 그 안의 정보들을 반환하여 보여주는 역할을 합니다

nginx와 apache라는 친구들은 정적인 데이터를 처리하는 웹 서버입니다 apache 웹 서버의 단점을 보완하기 위해 만들어진 것이 nginx웹서버이죠

tomcat은 동적인 데이터를 처리하는 웹 서버(WAS)입니다 데이터베이스와 연결되어 데이터를 주고받는 웹 서버입니다

이 외에도 다양한 웹서버들이 존재하는데 각자의 차이를 알고 내가 필요한 친구를 골라서 목적에 맞게 사용하는 것이 가장 바람직합니다

HTML !
HTML은 HyperText Markup Language의 약자입니다.

<알면 좋은 개념>
웹 페이지는 HTML 문서라고도 불리며, HTML 태그들로 구성됩니다.

각각의 HTML 태그는 웹 페이지의 디자인이나 기능을 결정하는데 사용됩니다.

모든 태그는 미리 정의되어 있으며, 각각의 태그와 속성을 사용하기만 하면 됩니다.
CSS !
CSS는 Cascading Style Sheets의 약자입니다.

CSS는 HTML 요소들이 각종 미디어에서 어떻게 보이는가를 정의하는 데 사용되는 스타일 시트 언어입니다.

웹 페이지의 스타일과 내용적인 부분을 서로 분리해 놓은 서식을 스타일 시트라고 합니다. 
이러한 스타일 시트를 이용하면 웹 페이지의 스타일을 편리하게 개발할 수 있습니다..
HTML이 건물의 뼈대를 세우는 일이였다면, CSS는 건물을 이쁘게 꾸밀 수 있는 언어입니다!
JavaScript !
자바스크립트(JavaScript)는 객체(object) 기반의 스크립트 언어입니다.

HTML로는 웹의 내용을 작성하고, CSS로는 웹을 디자인하며, 자바스크립트로는 웹의 동작을 구현할 수 있습니다. 
자바스크립트는 주로 웹 브라우저에서 사용되나, Node.js와 같은 프레임워크를 사용하면 서버 측 프로그래밍에서도 사용할 수 있습니다.

컴퓨터나 스마트폰 등에 포함된 대부분의 웹 브라우저에는 자바스크립트 인터프리터가 내장되어 있습니다.
자바와 자바스크립트는 그 이름만 놓고 보면 서로 관련이 있는 언어로 생각되기 쉽습니다.
하지만 두 언어는 서로 직접적인 관련은 없으며, 비슷한 점보다는 다른 점이 훨씬 많습니다.
건물을 이쁘게 꾸몄다면, 이제 건물의 기능을 만들어 생동감 있게 연출합니다!
React !
사용자와 상호작용할 수 있는 동적인 웹 사이트를 만들려고 하면 어떻게 해야 할까요? 바로 React를 사용하는 방법이 있습니다.

React.js는 사용자 인터페이스를 구축하는 데 사용되는 오픈 소스 JavaScript 라이브러리입니다.
웹 및 모바일 앱의 view layer를 처리하는 데 사용됩니다. 또한 React를 사용하면 재사용 가능한 UI 구성 요소를 만들 수 있습니다.

DBMS를 통해 관리하며 학교에서는 관계형 데이터베이스를 위주로 배우며 
React를 사용하면 개발자가 페이지를 다시 로드하지 않고도 데이터를 변경할 수 있는 대규모 웹 애플리케이션을 만들 수 있습니다.
React의 주요 목적은 빠르고 확장 가능하며 단순합니다.
